# Scope of This Model

This section defines the scope, assumptions, and included components of the final state space estimation.

* Minecraft Java Edition **1.21.11**

  * Game engine logic
  * Memory representation
  * NBT data

  * Any other mechanically detectable state

---

# Fundamental Assumptions

This model is based on the following assumptions:

* Any difference the game can detect or store is a valid state difference.
* Only differences that are *mechanically distinguishable by the game* are counted.
* Purely visual or client-only differences are ignored unless they affect game logic.
* The system is treated as a finite, discrete state space.

---

# Core Rule Set

* “Infinity” is excluded as a modeling construct.
* If a difference can be represented or detected by the game engine (including memory, NBT, redstone logic, packets, or commands), it is treated as a distinct state.
* Purely visual differences that do not affect game logic are not counted as separate states.
